# ETF 추천·분석 웹앱 — 작업 규칙 (Claude Code용)

미국 ETF 추천·분석 웹앱. 사용자가 투자 성향/목표를 입력하면 실시간 데이터로 ETF를 추천·분석한다. 나와 배우자가 함께 쓰는 개인용 앱이며, 핸드폰에서도 잘 보여야 한다.

자세한 현황은 `작업노트.md` 참고.

## 기술 스택 (바꾸지 말 것)
- 백엔드: Python + FastAPI → `main.py` 단일 파일
- 데이터: yfinance (실패 시 폴백 기본값, `source` 표시)
- 프론트: `static/index.html` 단일 파일 + Chart.js
- 포트 8000. 요청 없으면 새 프레임워크 도입 금지.

## 반드시 지킬 원칙
1. 모든 주석·UI 문구·설명은 한국어.
2. 정직함이 최우선 — 설명·숫자는 코드가 실제로 쓰는 계산식과 1:1 일치. 지어내지 말 것. 가정(세금·수수료·환율 미반영 등)은 항상 명시.
3. yfinance 실패해도 앱이 죽으면 안 됨 — 항상 폴백하고 `source: "fallback"` 표시.
4. 교육·참고용 전제. 투자 권유 표현 금지.
5. 디자인은 Apple 톤 — 단색 액션블루(#0066cc), Inter 폰트, 넉넉한 여백, 그림자 최소, 알약형 버튼. 두 번째 강조색·그라데이션 금지.
6. 반응형 필수 — PC는 좌우 배치, 핸드폰은 위아래 세로 배치 자동 전환. 한 화면에서 선택과 결과를 같이.

## 사용성 원칙
- 추천 항목은 처음부터 전체 선택 상태로 결과를 바로 보여주고, 체크 해제/추가 시 실시간 갱신.
- "왜 이게 추천인지" 설명을 항상 함께 표시.
- 페이지 로드 시 자동으로 무거운 결과를 띄우지 말 것 (의도한 액션에만 반응).
- 종목별 Yahoo Finance(finance.yahoo.com/quote/{티커}) 새 탭 링크 제공.

## 작업 방식
- 큰 변경은 기능 단위로 나눠서 진행.
- 기능/계산 로직 변경과 디자인만 바꾸는 변경을 섞지 말 것.
- 변경 후 무엇을 왜 바꿨는지 한국어로 간단히 설명.
- 실제 동작은 서버 띄워서 검증.

## 실행
```
pip install -r requirements.txt
python main.py
```
브라우저에서 http://127.0.0.1:8000 접속.

## 알려진 이슈
- 회사망(SSL 검사)에서 yfinance가 `SSL certificate problem: unable to get local issuer certificate`로 실패할 수 있음 → 현재는 자동으로 폴백(기본값)으로 동작. 실시간 연동을 위한 인증서 처리 방법은 추후 작업.
