# ETF 추천·분석 웹앱 — 작업 노트

개인용(부부 사용) 미국 ETF 추천·분석 웹앱. 위험 성향과 목표를 고르면 대표 미국 ETF를 점수화해 보여준다. 교육·참고용이며 투자 권유가 아니다.

최종 업데이트: 2026-06-22

---

## 1. 구성

| 구분 | 내용 |
|------|------|
| 백엔드 | Python + FastAPI, 단일 파일 `main.py` |
| 데이터 | yfinance (실패 시 종목별 폴백값 + `source` 표시) |
| 프론트 | `static/index.html` 단일 파일 + Chart.js |
| 포트 | 8000 |

```
ETF/
├─ main.py              # FastAPI 서버 + 추천 로직
└─ static/
   └─ index.html        # 화면 전체 (HTML+CSS+JS 단일 파일)
```

## 2. 만든 기능

### 입력
- 위험 성향: 안정형 / 중립형 / 공격형 (알약 버튼)
- 투자 목표: 성장 / 배당 / 분산 (알약 버튼)
- 분석할 종목: 대표 미국 ETF 31개 체크박스 (처음엔 전체 선택)

### 동작
- "추천 받기"를 눌러야 결과가 뜬다 (페이지 로드 시 자동으로 무거운 계산을 돌리지 않음).
- 첫 실행 이후에는 성향·목표·종목 체크를 바꾸면 결과가 실시간으로 갱신된다.
- 종목별로 "왜 추천인지" 한국어 설명과 Yahoo Finance 새 탭 링크 제공.
- 결과는 점수순 정렬 + 막대 차트.

### 추천 대상 풀 (31개)
- 성장: VOO, VTI, QQQ, VUG, SCHG
- 배당: SCHD, VYM, DGRO, HDV, VIG
- 채권: BND, AGG, TLT, SHY, BNDX
- 분산: VT, VEA, VWO, VNQ, GLD
- 섹터: XLK(기술), XLF(금융), XLV(헬스케어), XLE(에너지), XLY(임의소비재), XLP(필수소비재), XLI(산업재), XLU(유틸리티), XLB(소재), XLRE(리츠), XLC(커뮤니케이션서비스) — 미국 11개 GICS 섹터 SPDR Select Sector ETF

> 섹터 11종의 yield/ret1y 폴백값은 stockanalysis.com 조회값(2026-06-19 기준). vol 폴백값은 직접 제공되지 않아 베타×시장변동성(VOO 기준 ≈17%)을 기본으로, 베타가 못 잡아내는 섹터 특성(예: 에너지의 원자재 변동성, 유틸리티/필수소비재의 낮은 변동성, 기술/임의소비재의 높은 변동성)을 반영해 추정. yfinance가 살아있으면 이 폴백 대신 실데이터를 쓴다.

## 3. 점수 계산식 (화면 설명과 1:1 일치)

종목별 지표(yfinance로 계산, 실패 시 폴백):
- 1년 수익률(%) — 최근 1년 가격 변화
- 변동성(%) — 일간수익률 표준편차 × √252 (연 환산)
- 배당수익률(%) — 최근 12개월 배당 합계 ÷ 현재가

세부점수(0~100):
- 수익성 R = (1년수익률 + 10) / 50 × 100
- 배당   D = 배당수익률 / 5 × 100
- 안정성 S = (35 − 변동성) / 35 × 100

목표별 기본 가중치:

| 목표 | 수익성 | 배당 | 안정성 |
|------|-------|------|-------|
| 성장 | 0.60 | 0.10 | 0.30 |
| 배당 | 0.20 | 0.60 | 0.20 |
| 분산 | 0.34 | 0.33 | 0.33 |

위험 성향 보정 (shift = 0.20):
- 안정형: 안정성 +0.20 (수익성·배당에서 0.10씩 차감)
- 중립형: 변화 없음
- 공격형: 수익성 +0.20, 안정성 −0.20
- 보정 후 음수는 0으로 자르고 합이 1이 되도록 재정규화

최종점수 = 수익성×wR + 배당×wD + 안정성×wS (0~100)

> 가정: 세금·매매수수료·환율은 반영하지 않는다. 모든 수치는 교육·참고용.

## 5. 안정성 처리

- yfinance가 실패하거나 네트워크가 막혀도 앱이 죽지 않고 종목별 폴백값을 쓴다. 화면 우측 상단 배지로 항상 구분: `실시간 (yfinance)` / `기본값 (fallback)` / `일부 기본값 (mixed)`.
- yfinance 호출에 timeout(5초)을 둬 느릴 때 빠르게 폴백한다.
- 프론트는 서버 연결 실패나 `file://` 직접 열기를 감지해 원인과 해결법을 한국어 배너로 안내하고, 그래도 종목 목록은 비상용 내장 목록으로 항상 표시한다.
- Chart.js 로드 실패 시에도 카드·수치는 정상 표시된다.

---

## 6. 고친 문제

증상: 종목 목록이 안 나오고 "추천 받기"가 작동 안 함.

원인: 프론트 JS에 에러 처리가 없어, 파일을 직접 열거나(`file://`) 서버 미실행 등 한 가지라도 실패하면 목록을 못 불러오고 안내 없이 멈췄다. 체크박스가 비면 선택 종목이 없어 추천도 동작하지 않았다. (서버 자체는 정상 확인됨.)

조치: 위 5번의 안정성 처리를 추가. 비상용 목록 내장, 연결 실패·`file://` 감지 배너, 호출 실패 메시지, Chart.js 방어 코드.

## 7. 실행 방법

```bash
pip install fastapi uvicorn yfinance
python main.py          # 또는: uvicorn main:app --port 8000
```

브라우저에서 `http://127.0.0.1:8000` 접속. (파일을 직접 더블클릭하면 작동하지 않음 — 반드시 서버를 켜고 주소로 접속.)

### 핸드폰에서 보기 (같은 WiFi)
1. PC에서 서버 실행 (`0.0.0.0:8000`으로 열려 폰 접속 가능).
2. PC 내부 IP 확인: 명령창에서 `ipconfig` → IPv4 주소 (예: 192.168.0.10).
3. 폰 브라우저에서 `http://192.168.0.10:8000` 접속.

### 집 밖에서도 접속 (인터넷 배포 — Render 무료)
PC를 꺼도 폰에서 언제든 접속하려면 클라우드에 올린다. 저장소에 배포 설정을 포함해 둠:
- `Procfile`, `render.yaml` : 배포 명령(호스트가 주입하는 `$PORT` 사용).
- `main.py` 는 `PORT` 환경변수를 읽어 어떤 호스트에서도 동작.

배포 순서:
1. 이 저장소를 GitHub에 푸시(이미 되어 있음).
2. https://render.com 가입 → New → Web Service → 이 저장소 연결.
3. `render.yaml` 을 자동 인식해 빌드·실행 → `https://etf-recommender-XXXX.onrender.com` 주소가 생김.
4. 그 주소를 폰/PC 어디서나 열면 됨. 폰 홈화면에 추가하면 앱처럼 쓸 수 있음.

참고: Render 무료 플랜은 일정 시간 미사용 시 잠들었다가 첫 접속에서 ~30초 깨어남(이후엔 빠름).

### 접속 비밀번호 (선택)
- 환경변수 `APP_PASSWORD` 를 설정하면 모든 페이지·API 에 비밀번호 로그인을 요구한다.
  - 설정 안 하면(로컬 개발 등) 로그인 없이 그대로 동작.
  - 배포(공개) 시에는 반드시 설정할 것. Render는 `render.yaml`에 `APP_PASSWORD`를
    `sync:false`로 선언해 둬, 대시보드에서 값을 입력하면 됨(깃에 저장 안 됨).
- 동작: `/login` 한국어 로그인 화면 → 맞으면 HMAC 서명 쿠키(HttpOnly, HTTPS면 Secure,
  30일 유지) 발급. 비밀번호 자체는 쿠키에 담지 않음. 비밀번호를 바꾸면 기존 쿠키는 무효화.
  `/logout` 으로 로그아웃.
- 표준 라이브러리(hmac/hashlib)만 사용 — 추가 의존성 없음.
- 한계: 단일 공유 비밀번호(부부 공용)다. 사용자별 계정·비번 변경 UI는 없음.

## 8. 검증 현황

- 백엔드: 실제 HTTP로 `/`(200), `/api/etfs`(31개, 5개 카테고리 — 성장/배당/채권/분산 5개씩 + 섹터 11개) 정상.
- 섹터 추가 검증: `/api/recommend`에 섹터 티커만 넘겨 성장+공격형으로 호출 → XLK(1년수익률 59.6%)가 1위로 정렬, 점수·근거 문구 모두 기존 공식 그대로 계산됨을 확인. yfinance가 막힌 환경에서도 전 종목 폴백으로 전환되며 앱이 죽지 않음을 재확인(`source: fallback`).
- 점수: 9개(성향×목표) 가중치 합 = 1.0 확인. 배당+안정형 → 채권·배당 ETF 상위, 성장+공격형 → 성장 ETF 상위로 의도대로 정렬.
- 프론트: 화면 JS 전체 문법 검사 통과, 파일 온전함 확인. `POOL_FALLBACK`(비상용 내장 목록)에도 섹터 11종 반영.
- 미검증: 이 작업 환경은 브라우저 설치가 막혀 실제 브라우저 클릭까지의 검증은 못 함 → PC에서 한 번 확인 필요.
